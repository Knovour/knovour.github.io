React/Redux 一個月的開發感想

2016-05-21
react.js, redux

之前自學 React 時寫了一個簡單的線上簡歷，而我也確實靠他加分而拿到了工作，之後進去也是開始用 React 建立頁面，之後加上了 Redux；雖然這兩個總天數加起來大概就一個月，不敢說自己有摸清這些東西，不過還是就開發經驗來寫一些感想。

先講結論

喜歡嗎？

其實不討厭，還算是喜歡的；但若是要我向別人推荐的話，我會建議學 Vue.js 入門，熟了再考慮要不要碰 React。若是不排斥用其他語言轉換成 js 的話，Elm 看起來是個不錯的選擇，但 ES6 開始普及之後我對這方面的興趣不大，所以這個建議比較不負責任一點。

網路上的範例能幫到多少？

在我決定 ES6 + React 一次學的時候，卻發現網路上的範例已經開始塞 ES7 在裏面，在沒有大致摸過 React 跟 Redux 的狀況下，看到有塞 ES7 語法加上許多 plugins 的範例時，這幫助實在不大。

開發過程的感受如何？

摸新東西當然都會面臨挫折、觀念轉變以及學習曲線的問題，但老實說，不把前後分離這件事考慮進去的話（畢竟這是前端 framework 最主要的目的），失去的不會比獲得的感受來的少多少，頂多接近打平，剩下的端看你的 M 體質點多高來去彌補。說實在的，當初把玩 Rust 語言被摧殘到躺地板時我覺得還比較爽一點。

學習與開發重點

ES6

Airbnb JavaScript Style Guide 可以當成一個不錯的 ES6 入門文件，之後再閱讀 ECMAScript 6 入门做更進一步的了解。

React

  • 能用 css 解決的就用 css。
  • 能用 Virtual DOM 解決的就不要用原生操作。
  • Say goodbye to jQuery.
  • Airbnb React/JSX Style Guide 在 React 稍微懂一些後可以參考看看。
  • React 大部分的 plugins 程式碼都不多，使用前不妨看一下原始碼，或許你只需要擷取出一部份，不用再多載一個就能自己實現。
  • 有一定程度了解後，ES7 不妨給他用下去。

Redux

  • 如果只是改變 component 自身狀態，不要考慮用。
  • 跨 component 的事件處理如果數量不多，可以考慮不用。
  • 第一次用建議不要用太多額外的 plugins，先知道基本用法是怎樣，之後再去考慮這件事。

後記

我想我下個階段會研究 RxJS 與 Cycle.js 看看吧。

線上課程

  • React.js Program: React 線上教學，其中 React.js Fundamentals 這個入門課程是免費的，推荐。
  • Getting Started with Redux: Redux 入門課程。
